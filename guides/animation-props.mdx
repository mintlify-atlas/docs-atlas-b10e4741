Squiggly Sliders uses animated sine waves to create its distinctive wave effect. You can customize the animation behavior using amplitude, wavelength, and speed properties.

## Understanding the wave animation

The slider draws two animated sine waves:

- **Active wave** - The filled portion from the start to the current value
- **Passive wave** - The unfilled portion from the current value to the end

Each wave has independent control over its amplitude (height) and wavelength (frequency).

## Amplitude

Amplitude controls the wave height (how far up and down the wave oscillates).

### Active amplitude

Controls the wave height for the filled portion:

```svelte
<script>
  import SquigglySlider from 'squiggly-sliders';
</script>

<SquigglySlider
  min={0}
  max={10}
  value={6}
  activeAmplitude={7}
  active="--md-sys-color-primary"
  passive="--md-sys-color-on-surface"
/>
```

- **Range**: 0-7
- **Default**: 4
- **Effect**: Higher values create taller waves

<Info>
`activeAmplitude={0}` creates a straight line with no wave effect.
</Info>

### Passive amplitude

Controls the wave height for the unfilled portion:

```svelte
<script>
  import SquigglySlider from 'squiggly-sliders';
</script>

  min={0}
  max={10}
  value={6}
  passiveAmplitude={2}
  active="--md-sys-color-primary"
  passive="--md-sys-color-on-surface"
/>
```

- **Range**: 0-7
- **Default**: 2
- **Effect**: Higher values create taller waves

## Wavelength

Wavelength controls the wave frequency (how many complete wave cycles fit in a given space).

### Active wavelength

Controls the wave frequency for the filled portion:

```svelte
<script>
  import SquigglySlider from 'squiggly-sliders';
</script>

<SquigglySlider
  min={0}
  max={10}
  value={6}
  activeWavelength={2}
  active="--md-sys-color-primary"
  passive="--md-sys-color-on-surface"
/>
```

- **Range**: 0.25-7
- **Default**: 1
- **Effect**: Higher values create wider, more stretched waves

- **Range**: 0.25-7

## Speed factor

Speed factor controls how fast the waves animate horizontally:

```svelte
<script>
  import SquigglySlider from 'squiggly-sliders';
</script>

<SquigglySlider
  min={0}
  max={10}
  value={6}
  speedFactor={8}
  active="--md-sys-color-primary"
  passive="--md-sys-color-on-surface"

```

- **Range**: 1-8
- **Default**: 4
- **Effect**: Higher values create faster animation

## How the animation works

The wave animation uses the sine function with the following formula:

```javascript
y = amplitude * Math.sin(x / (6 * wavelength) + time / ((10 - speedFactor) * 60)) + centerY
```

Where:
- `amplitude` is `activeAmplitude` or `passiveAmplitude`
- `wavelength` is `activeWavelength` or `passiveWavelength`
- `speedFactor` affects the time component
- `centerY` is the vertical center of the canvas

<Info>
The animation runs at 60fps using `requestAnimationFrame` for smooth, efficient rendering.
</Info>

## Performance considerations

The animation runs continuously using `requestAnimationFrame`. For optimal performance:

- Use moderate amplitude values (2-5 range)
- Avoid extreme wavelength values at the ends of the range
- Consider pausing animation when the slider is not visible

<Warning>
Very high amplitude values (6-7) combined with very low wavelength values (0.25-0.5) can create dense, complex animations that may impact performance on lower-end devices.
</Warning>

## Pausing and resuming animation (Web Component only)

The Web Component provides methods to control animation:

```javascript
const slider = document.querySelector('squiggly-slider');

// Pause animation
slider.pause();

// Resume animation
slider.resume();
```

<Info>
Pausing the animation when the slider is not visible (e.g., off-screen or in a hidden tab) can improve overall application performance.
</Info>

## Animation property ranges

Quick reference for all animation property ranges:

| Property | Minimum | Maximum | Default |
|----------|---------|---------|----------|
| `activeAmplitude` | 0 | 7 | 4 |
| `passiveAmplitude` | 0 | 7 | 2 |
| `activeWavelength` | 0.25 | 7 | 1 |
| `passiveWavelength` | 0.25 | 7 | 1 |
| `speedFactor` | 1 | 8 | 4 |

<Note>
These ranges are recommendations based on the source code. You can technically use values outside these ranges, but they may produce unexpected visual results.
</Note>
