Squiggly Sliders provides a native Svelte component with full two-way binding support. This guide shows you how to use it in your Svelte applications.

## Installation

Install the package using your preferred package manager:

<CodeGroup>
```bash npm
npm i squiggly-sliders
```

```bash pnpm
pnpm add squiggly-sliders
```

```bash yarn
yarn add squiggly-sliders
```
</CodeGroup>

## Basic usage

Import the component and use it with basic props:

```svelte
<script>
  import SquigglySlider from 'squiggly-sliders';
</script>

<SquigglySlider
  min={0}
  max={10}
  value={6}
  active={'--md-sys-color-primary'}
  passive={'--md-sys-color-on-surface'}
/>
```

<Note>
The component reads colors from CSS variables by default, making it easy to integrate with your existing design system.
</Note>

## Controlled component with bind:value

You can use Svelte's two-way binding to create a controlled component:

```svelte
<script>
  import SquigglySlider from 'squiggly-sliders';
  let value = 6;
</script>

<SquigglySlider
  bind:value
  min={0}
  max={10}
  step={1}
  active={'--md-sys-color-primary'}
  passive={'--md-sys-color-on-surface'}
  activeAmplitude={4}
  passiveAmplitude={2}
  activeWavelength={1}
  passiveWavelength={1}
  speedFactor={4}
/>

<p>Current value: {value}</p>
```

<Info>
Using `bind:value` creates a two-way binding where changes to the slider update your variable, and changes to your variable update the slider.
</Info>

## Uncontrolled component

For simpler use cases where you don't need to track the value in your script, you can use the component without binding:

```svelte
<script>
  import SquigglySlider from 'squiggly-sliders';
</script>

<SquigglySlider
  min={0}
  max={10}
  value={6}
  active={'--md-sys-color-primary'}
  passive={'--md-sys-color-on-surface'}
/>
```

## Customizing animation

You can customize the wave animation using amplitude, wavelength, and speed properties:

```svelte
<script>
  import SquigglySlider from 'squiggly-sliders';
  let value = 5;
</script>

<SquigglySlider
  bind:value
  min={0}
  max={10}
  activeAmplitude={7}
  passiveAmplitude={3}
  activeWavelength={2}
  passiveWavelength={1.5}
  speedFactor={6}
  active={'--md-sys-color-primary'}
  passive={'--md-sys-color-on-surface'}
/>
```

## All available props

<Tabs>
  <Tab title="Range props">
    - `min` - Minimum value (default: `0`)
    - `max` - Maximum value (default: `10`)
    - `step` - Step increment (default: `1`)
    - `value` - Current value
    - `title` - Accessible title for the slider
  </Tab>
  
  <Tab title="Color props">
    - `active` - CSS variable name for active portion (default: `'--md-sys-color-primary'`)
    - `passive` - CSS variable name for passive portion (default: `'--md-sys-color-on-surface'`)
  </Tab>
  
  <Tab title="Animation props">
    - `activeAmplitude` - Wave height for active portion, range 0-7 (default: `4`)
    - `passiveAmplitude` - Wave height for passive portion, range 0-7 (default: `2`)
    - `activeWavelength` - Wave frequency for active portion, range 0.25-7 (default: `1`)
    - `passiveWavelength` - Wave frequency for passive portion, range 0.25-7 (default: `1`)
    - `speedFactor` - Animation speed, range 1-8 (default: `4`)
  </Tab>
</Tabs>

## Responsive behavior

The component automatically resizes when the window changes. The canvas width adjusts to match its parent container:

```svelte
<script>
  import SquigglySlider from 'squiggly-sliders';
  let value = 5;
</script>

<div style="width: 100%; max-width: 600px;">
  <SquigglySlider
    bind:value
    min={0}
    max={100}
    active={'--md-sys-color-primary'}
    passive={'--md-sys-color-on-surface'}
  />
</div>
```

<Note>
The component listens to window resize events and automatically adjusts the canvas dimensions.
</Note>

## Accessibility

The component uses a native HTML range input underneath the canvas, ensuring keyboard accessibility:

```svelte
<SquigglySlider
  bind:value
  min={0}
  max={10}
  title="Volume control"
  active={'--md-sys-color-primary'}
  passive={'--md-sys-color-on-surface'}
/>
```

<Info>
The `title` prop sets an accessible label for screen readers. The canvas element is marked with `aria-hidden="true"` so assistive technology focuses on the underlying range input.
</Info>

## Dark mode support

The component automatically responds to color scheme changes by re-reading CSS variables:

```svelte
<script>
  import SquigglySlider from 'squiggly-sliders';
  let value = 5;
</script>

<SquigglySlider
  bind:value
  min={0}
  max={10}
  active={'--md-sys-color-primary'}
  passive={'--md-sys-color-on-surface'}
/>

<style>
  :global(:root) {
    --md-sys-color-primary: rgb(97 97 24);
    --md-sys-color-on-surface: rgb(28 28 20);
  }

  @media (prefers-color-scheme: dark) {
    :global(:root) {
      --md-sys-color-primary: rgb(220 220 140);
      --md-sys-color-on-surface: rgb(230 230 220);
    }
  }
</style>
```

<Note>
The component listens to `prefers-color-scheme` media query changes and automatically updates colors.
</Note>
